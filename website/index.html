How it works

  1. On the receiving machine, start receive mode and choose a save folder.
  2. On the sending machine, pick a file and select the receiver from the LAN device list.
  3. The receiver accepts or rejects the transfer before any file bytes are sent.
  4. LFT verifies the file with SHA-256 when the transfer finishes.

CLI (available now)

The command-line tool is ready for same-network transfers:

# Receiver
lft recv --out ./downloads/

# Sender — discover by device name
lft list
lft send --to "Device-Name" --file ./video.mp4

# Fallback — manual IP when discovery is blocked
lft send --host 192.168.1.42 --file ./video.mp4

Build from source

git clone https://github.com/Parkryan0128/LFT.git
cd LFT
brew install libmsquic openssl@3 cmake
cmake -S . -B build
cmake --build build
./build/src/lft_cli --help

See the GitHub repository for full build instructions and dependencies.

Limitations

  • Same local network only — not for sending files over the internet.
  • One file per transfer. Zip folders if you need more.
  • Both devices must be running LFT.
  • Some guest or corporate Wi‑Fi networks block device discovery; use manual IP.
